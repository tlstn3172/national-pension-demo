**프로젝트명:** My Pension Rescue (MPR)  
**버전:** 1.0  
**작성일:** 2025-12-23  
**문서 상태:** Draft

## 1. 개요 (Executive Summary)

### 1.1 프로젝트 배경

국민연금 고갈 우려는 대한민국 사회의 가장 중요한 정책 이슈 중 하나입니다. 현재 보험료율 9%, 소득대체율 40% 체계를 유지할 경우 기금 고갈이 예상되며, 이는 세대 간 불신과 정책 개혁에 대한 막연한 공포를 심화시키고 있습니다.

### 1.2 문제 정의 (Problem Statement)

- **핵심 문제:** 복잡한 연금 정책 변수(보험료율, 소득대체율, 수급연령 등)에 대한 이해 부족으로 인한 감정적 논쟁 심화
- **기회:** 정책 변수를 직접 조정하며 기금 고갈 시점의 변화를 직관적으로 이해할 수 있는 Interactive Tool의 부재
- **영향:** 합리적인 정책 논의 저해 및 세대 간 갈등 증폭

### 1.3 솔루션

데이터 기반의 인터랙티브 시뮬레이션 플랫폼을 통해 사용자가 직접 정책 변수를 조정하고, 그 결과를 실시간으로 확인하며 현실적인 '지속 가능한 연금 모델'을 탐색할 수 있는 환경을 제공합니다.

### 1.4 목표

- 복잡한 연금 정책을 시각적으로 이해하기 쉽게 표현
- 데이터 기반의 합리적인 정책 논의 촉진
- 세대 간 형평성을 고려한 지속 가능한 연금 모델 탐색 지원

## 2. 타겟 유저 (Target Audience)

### 2.1 Primary Users

#### 2030 세대
- **특징:** 본인이 낼 보험료와 받을 연금의 현실적인 수치를 확인하고 싶은 예비 가입자
- **니즈:** 미래 연금 수급액 예측, 보험료 부담 시뮬레이션
- **페인 포인트:** 복잡한 연금 제도에 대한 이해 부족, 미래 불확실성

#### 정책 관심층
- **특징:** 연금 개혁의 논리를 수치로 검증해 보고 싶은 시민
- **니즈:** 다양한 개혁안의 효과 비교, 데이터 기반 정책 이해
- **페인 포인트:** 정책 정보의 접근성 부족, 복잡한 수식과 통계

### 2.2 Secondary Users

#### 포트폴리오 평가자
- **특징:** 복잡한 수식을 제품화하고 전략적 대안을 제시하는 기획 역량을 확인하려는 채용 담당자
- **니즈:** 문제 해결 능력, 데이터 기반 의사결정 역량 확인
- **평가 포인트:** 기획 역량, 기술적 구현 능력, 전략적 사고

## 3. 핵심 기능 (Key Features)

### 3.1 대시보드: "Current Status"

#### 기능 설명
현재 법령 기준(보험료율 9%, 소득대체율 40%) 유지 시 기금 소진 연도 및 예상 수급액을 시각화합니다.

#### 주요 구성 요소
- **기금 잔액 추이 그래프:** 연도별 국민연금 기금 잔액 변화를 Area Chart로 표시
- **고갈 예상 시점:** 현재 정책 유지 시 기금 소진 예상 연도 강조 표시
- **주요 지표 카드:**
  - 현재 보험료율: 9%
  - 현재 소득대체율: 40%
  - 현재 수급 연령: 65세
  - 기금 운용 수익률: 역사적 평균값

#### 데이터 소스
- 통계청 장래인구추계
- 국민연금공단 공시 자료
- KOSIS API 연동

### 3.2 시나리오 시뮬레이터: "Reform Lab"

#### 기능 설명
사용자가 정책 변수를 직접 조정하여 다양한 개혁 시나리오의 효과를 실시간으로 시뮬레이션합니다.

#### 조정 가능한 변수

| 변수 | 현재값 | 조정 범위 | 설명 |
|------|--------|-----------|------|
| **보험료율** | 9% | 9% ~ 15% | 매월 소득에서 내는 연금 보험료 비율 |
| **소득대체율** | 40% | 40% ~ 50% | 생애 평균 소득 대비 연금액 비율 |
| **수급 연령** | 65세 | 65세 ~ 70세 | 연금을 받기 시작하는 나이 |
| **기금 운용 수익률** | 5% | 3% ~ 7% | 연금 기금 투자 수익률 |

#### 핵심 로직

```
기금 잔액(t+1) = 기금 잔액(t) × (1 + 수익률) + 보험료 수입(t) - 연금 지급액(t)

보험료 수입(t) = 가입자 수(t) × 평균 소득(t) × 보험료율

연금 지급액(t) = 수급자 수(t) × 평균 소득(t-n) × 소득대체율
```

#### 시각화 요소
- **비교 그래프:** 현재 정책 vs 조정된 시나리오의 기금 잔액 추이 비교
- **고갈 시점 변화:** 정책 조정에 따른 기금 고갈 시점 변화 표시
- **개인 영향 분석:** 사용자 연령대별 예상 보험료 부담 및 수급액 변화

### 3.3 세대 간 부담 리포트: "Sustainability Insight"

#### 기능 설명
설정한 시나리오가 채택될 경우, 미래 세대가 감당해야 할 부담을 분석하여 정책의 형평성을 평가합니다.

#### 주요 지표

**지속가능성 점수 (Sustainability Score)**
- 기금 고갈 방지 여부: 40점
- 세대 간 부담 형평성: 30점
- 경제적 실현 가능성: 30점
- 총점: 100점 만점

**세대별 영향 분석**
- 1980년대생: 보험료 부담 변화, 예상 수급액
- 1990년대생: 보험료 부담 변화, 예상 수급액
- 2000년대생: 보험료 부담 변화, 예상 수급액
- 2010년대생 이후: 조세 부담률, 재정 건전성 영향

#### 인사이트 제공
- **경고 메시지:** 특정 세대에 과도한 부담이 집중되는 경우 알림
- **추천 조정:** 형평성을 개선할 수 있는 변수 조정 제안
- **비교 분석:** 다른 시나리오와의 형평성 비교

## 5. 성공 지표 (Success Metrics)

### 5.1 사용자 참여도 (Engagement)

- **시나리오 비교 횟수:** 유저가 평균 3개 이상의 시나리오를 비교 분석
- **세션 시간:** 평균 세션 시간 5분 이상
- **재방문율:** 7일 내 재방문율 30% 이상

### 5.2 사용성 (Usability)

- **직관성:** 복잡한 연금 구조를 직관적인 그래프(Area Chart 등)로 표현
- **이해도:** 사용자 설문을 통한 정책 이해도 향상 측정
- **만족도:** 사용자 만족도 점수(CSAT) 4.0/5.0 이상

### 5.3 전략적 가치 (Strategic Value)

- **최적 타협점 제안:** 단순 시뮬레이션을 넘어, 기획자로서 '최적의 타협점(Golden Ratio)' 제안
- **인사이트 품질:** 세대 간 형평성 분석의 정확성 및 유용성
- **포트폴리오 효과:** 채용 담당자의 긍정적 피드백 수집

### 5.4 기술적 품질 (Technical Quality)

- **성능:** 페이지 로드 시간 3초 이내
- **정확성:** 계산 로직의 수리적 정확성 검증
- **안정성:** 에러율 1% 미만

## 7. 리스크 및 제약사항 (Risks & Constraints)

### 7.1 데이터 관련 리스크

- **데이터 정확성:** 공개된 통계 데이터의 정확성 및 최신성 의존
- **완화 방안:** 신뢰할 수 있는 공식 출처(통계청, 국민연금공단) 사용, 데이터 출처 명시

### 7.2 기술적 리스크

- **계산 복잡도:** 복잡한 수리 모델의 정확한 구현 필요
- **완화 방안:** 전문가 검토, 단위 테스트 및 검증 프로세스 구축

### 7.3 사용자 경험 리스크

- **복잡성:** 너무 복잡한 UI로 인한 사용자 이탈
- **완화 방안:** 단계별 가이드, 툴팁, 튜토리얼 제공

### 7.4 정책적 리스크

- **정치적 민감성:** 연금 개혁은 정치적으로 민감한 주제
- **완화 방안:** 중립적 입장 유지, 데이터 기반 객관적 분석 강조

### 7.5 제약사항

- **개발 리소스:** 1인 개발 프로젝트로 인한 시간 및 리소스 제약
- **데이터 접근성:** 일부 상세 데이터는 공개되지 않을 수 있음
- **전문성:** 연금 정책 및 보험수리 전문 지식 필요
